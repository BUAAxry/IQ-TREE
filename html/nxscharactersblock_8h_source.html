00026 /*----------------------------------------------------------------------------------------------------------------------
00027 |       This class handles reading and storage for the NEXUS block CHARACTERS. It overrides the member functions Read and 
00028 |       Reset, which are abstract virtual functions in the base class NxsBlock. The issue of bookkeeping demands a careful
00029 |       explanation. Users are allowed to control the number of characters analyzed either by "eliminating" or "excluding"
00030 |       characters. Characters can be eliminated (by using the ELIMINATE command) at the time of execution of the data 
00031 |       file, but not thereafter. Characters can, however, be excluded at any time after the data are read. No storage is 
00032 |       provided for eliminated characters, whereas excluded characters must be stored because at any time they could be 
00033 |       restored to active status. Because one can depend on eliminated characters continuing to be eliminated, it would 
00034 |       be inefficient to constantly have to check whether a character has been eliminated. Hence, the characters are 
00035 |       renumbered so that one can efficiently traverse the entire range of non-eliminated characters. The original range 
00036 |       of characters will be hereafter denoted [0..`ncharTotal'), whereas the new, reduced range will be denoted 
00037 |       [0..`nchar'). The two ranges exactly coincide if `ncharTotal' = `nchar' (i.e., no ELIMINATE command was specified
00038 |       in the CHARACTERS block. The possibility for eliminating and excluding characters creates a very confusing situation
00039 |       that is exacerbated by the fact that character indices used in the code begin at 0 whereas character numbers in the
00040 |       data file begin at 1. The convention used hereafter will be to specify "character number k" when discussing 
00041 |       1-offset character numbers in the data file and either "character index k" or simply "character k" when discussing 
00042 |       0-offset character indices.
00043 |       
00044 |       There are several functions (and data structures) that provide services related to keeping track of the 
00045 |       correspondence between character indices in the stored data matrix compared to character numbers in the original 
00046 |       data file. The array `charPos' can be used to find the index of one of the original characters in the matrix. 
00047 |       The function GetCharPos provides public access to the protected `charPos' array. For example, if character 9 
00048 |       (= character number 10) was the only one eliminated, GetCharPos(9) would return UINT_MAX indicating that that 
00049 |       character 9 does not now exist. GetCharPos(10) returns 9 indicating that character 10 in the data file corresponds 
00050 |       to character 9 in the stored data matrix. All public functions in which a character number must be supplied (such 
00051 |       as GetInternalRepresentation) assume that the character number is the current position of the character in the data
00052 |       matrix. This allows one to quickly traverse the data matrix without having to constantly check whether or not a 
00053 |       character was eliminated. Note that GetNChar returns `nchar', not `ncharTotal', and this function should be used 
00054 |       to obtain the end point for a traversal of characters in the matrix. Other functions requiring a (current) character
00055 |       index are: 
00056 |>
00057 |       GetInternalRepresentation
00058 |       GetNumStates
00059 |       GetNumStates
00060 |       GetObsNumStates
00061 |       GetOrigCharIndex
00062 |       GetOrigCharNumber
00063 |       GetState
00064 |       HandleNextState
00065 |       HandleTokenState
00066 |       IsGapState
00067 |       IsMissingState
00068 |       IsPolymorphic
00069 |       ShowStateLabels
00070 |>
00071 |       The function IsEliminated is exceptional in requiring (by necessity) the original character index. The function 
00072 |       GetOrigCharIndex returns the original character index for any current character index. This is useful only when 
00073 |       outputting information that will be seen by the user, and in this case, it is really the character number that 
00074 |       should be output. To get the original character number, either add 1 to GetOrigCharIndex or call GetOrigCharNumber
00075 |       function (which simply returns GetOrigCharIndex + 1).
00076 |       
00077 |       A character may be excluded by calling the function ExcludeCharacter and providing the current character index or 
00078 |       by calling the function ApplyExset and supplying an exclusion set comprising original character indices. These 
00079 |       functions manipulate a bool array, `activeChar', which can be queried using one of two functions: IsActiveChar
00080 |       or IsExcluded. The array `activeChar' is `nchar' elements long, so IsActiveChar and IsExcluded both accept only 
00081 |       current character indices. Thus, a normal loop through all characters in the data matrix should look something 
00082 |       like this:
00083 |>
00084 |       for(unsigned j = 0; j < nchar; j++)
00085 |               {
00086 |               if (IsExcluded(j))
00087 |                       continue;
00088 |               .
00089 |               .
00090 |               .
00091 |               }
00092 |>
00093 |       A corresponding set of data structures and functions exists to provide the same services for taxa. Thus, `ntax'
00094 |        holds the current number of taxa, whereas `ntaxTotal' holds the number of taxa specified in the TAXA block. 
00095 |       If data is provided in the MATRIX command for all taxa listed in the TAXA block, ntax will be equal to `ntaxTotal'.
00096 |       If data is not provided for some of the taxa, the ones left out are treated just like eliminated characters. The 
00097 |       function GetTaxonPos can be used to query the `taxonPos' array, which behaves like the `charPos' array does for 
00098 |       characters: UINT_MAX for element `i' means that the taxon whose original index was `i' has been eliminated and no 
00099 |       data is stored for it in the matrix. Otherwise, GetTaxonPos(i) returns the current index corresponding to the taxon 
00100 |       with an original index of `i'. The function GetNTax returns `ntax', whereas GetNTaxTotal must be used to gain 
00101 |       access to `ntaxTotal' (but this is seldom necessary). The functions GetOrigTaxonIndex and GetOrigTaxonNumber behave 
00102 |       like their character counterparts, GetOrigCharIndex and GetOrigCharNumber. Like characters, taxa can be temporarily
00103 |       inactivated so that they do not participate in any analyses.until they are reactivated by the user. Inactivation 
00104 |       of a taxon is refered to as deleting the taxon, whereas restoring a taxon means reactivating it. Thus, the 
00105 |       ApplyDelset, DeleteTaxon, and RestoreTaxon functions correspond to the ApplyExset, ExcludeCharacter, and 
00106 |       IncludeCharacter functions for characters. To query whether a taxon is currently deleted, use either 
00107 |       IsActiveTaxon or IsDeleted. A normal loop across all active taxa can be constructed as follows:
00108 |>
00109 |       for (unsigned i = 0; i < ntax; i++)
00110 |               {
00111 |               if (IsDeleted(i))
00112 |                       continue;
00113 |               .
00114 |               .
00115 |               .
00116 |               }
00117 |>
00118 |       Below is a table showing the correspondence between the elements of a CHARACTERS block in a NEXUS file and the
00119 |       variables and member functions of the NxsCharactersBlock class that can be used to access each piece of information
00120 |       stored. Items in parenthesis should be viewed as "see also" items.
00121 |>
00122 |       NEXUS         Command        Data           Member
00123 |   Command       Atribute       Member         Functions
00124 |       ---------------------------------------------------------------------
00125 |       DIMENSIONS    NEWTAXA        newtaxa
00126 |       
00127 |                     NTAX           ntax           GetNTax

00129 |

00141 |                     SYMBOLS        symbols        GetSymbols
00142 |       
00143 |                     EQUATE         equates        GetEquateKey
00144 |                                                   GetEquateValue

00146 |       
00147 |                     MATCHCHAR      matchchar      GetMatchcharSymbol
00148 |       
00149 |                     (NO)LABELS     labels         IsLabels
00150 |       
00151 |                     TRANSPOSE      transposing    IsTranspose
00152 |       
00153 |                     INTERLEAVE     interleaving   IsInterleave

00157 |                     STATESFORMAT   (Note: only STATESPRESENT implemented)

00166 |                                                   GetNumStates

00169 |                                                   IsPolymorphic
00170 |>
00171 */

00429 /*----------------------------------------------------------------------------------------------------------------------
00430 |       Returns internal representation of the state for taxon `i', character `j'. In the normal situation, `k' is 0 meaning
00431 |       there is only one state with no uncertainty or polymorphism. If there are multiple states, specify a number in the 
00432 |       range [0..n) where n is the number of states returned by the GetNumStates function. Use the IsPolymorphic 
00433 |       function to determine whether the multiple states correspond to uncertainty in state assignment or polymorphism in 
00434 |       the taxon. The value returned from this function is one of the following:
00435 |~
00436 |       o -3 means gap state (see note below)
00437 |       o -2 means missing state (see note below)
00438 |       o an integer 0 or greater is internal representation of a state
00439 |~
00440 |       Note: gap and missing states are actually represented internally in a different way; for a description of the actual
00441 |       internal representation of states, see the documentation for NxsDiscreteDatum.
00442 */
00443 inline int NxsCharactersBlock::GetInternalRepresentation(
00444   unsigned i,   /* the taxon in range [0..`ntax') */
00445   unsigned j,   /* the character in range [0..`nchar') */
00446   unsigned k)   /* the 0-offset index of state to return */
00447         {
00448         if (IsGapState(i, j))
00449                 return -3;
00450         else if (IsMissingState(i, j))
00451                 return -2;
00452         else
00453                 return matrix->GetState(i, j, k);
00454         }
