00025 /*----------------------------------------------------------------------------------------------------------------------
00026 |       Class providing storage for the discrete data types (dna, rna, nucleotide, standard, and protein) inside a DATA or 
00027 |       CHARACTERS block. This class is also used to store the data for an ALLELES block. Maintains a matrix in which each 
00028 |       cell is an object of the class NxsDiscreteDatum. NxsDiscreteDatum stores the state for a particular combination of 
00029 |       taxon and character as an integer. Ordinarily, there will be a single state recorded for each taxon-character 
00030 |       combination, but exceptions exist if there is polymorphism for a taxon-character combination, or if there is 
00031 |       uncertainty about the state (e.g., in dna data, the data file might have contained an R or Y entry). Please consult 
00032 |       the documentation for the NxsDiscreteDatum class for the details about how states are stored. For data stored in an 
00033 |       ALLELES block, rows of the matrix correspond to individuals and columns to loci. Each NxsDiscreteDatum must 
00034 |       therefore store information about both genes at a single locus for a single individual in the case of diploid data.
00035 |       To do this, two macros HIWORD and LOWORD are used to divide up the unsigned value into two words. A maximum of 255 
00036 |       distinct allelic forms can be accommodated by this scheme, assuming at minimum a 32-bit architecture. Because it is
00037 |       not known in advance how many rows are going to be necessary, The NxsDiscreteMatrix class provides the AddRows 
00038 |       method, which expands the number of rows allocated for the matrix while preserving data already stored. 
00039 */
